# Codebase Overview: Full-Stack-Application-using-PHP-and-MySQL

## 1) What this project is

This repository is a classic PHP + MySQL monolithic web app for a **mobile-phone e-commerce store** ("Tech Store").
The application includes:

- Customer sign-up/sign-in.
- Product browsing by search and brand/category.
- Product detail pages.
- Cart and order placement.
- Contact form messages.
- Admin panel for brand/product CRUD and message viewing.

The stack is server-rendered PHP templates with inline SQL queries, plus CSS and a small amount of jQuery UI behavior.

---

## 2) High-level structure

- `htdocs/configuration/`
  - DB connection bootstrap (`configure.php`).
- `htdocs/log-in/`
  - Customer/admin auth entry points and logout handlers.
- `htdocs/files/`
  - Main user-facing pages (`search.php`, `Contact.php`, `ABOUT US.php`, etc.).
  - `Products/` product detail page.
  - `Categories/` brand listing page.
  - `cart/` cart pages and cart insert action.
  - `order/` order placement/list/remove action.
  - `Admin/` admin dashboard and brand/product management endpoints.
- `htdocs/css/`
  - Page/layout styles.
- `htdocs/jquery files/`
  - UI toggles for header categories and admin sections.
- `prjectdatabase.sql`
  - Full schema + seed data dump.

---

## 3) Runtime architecture

### Backend

- No framework/router; each `.php` file is both route handler and view template.
- Shared DB connection through `include_once("..\configuration\configure.php")` style includes.
- SQL executed via `mysqli_query()` directly inside page files.
- Session state via `$_SESSION` to track login context:
  - Customer session key: `log_in`.
  - Admin session key: `admin_in`.

### Frontend

- Mostly static HTML generated by PHP echo blocks.
- CSS files are global and page-specific (e.g., `product.css`, `cart.css`).
- jQuery scripts:
  - `header_category.js` controls category visibility in headers.
  - `admin.js` toggles admin panel sections.

### Database

Main tables in `prjectdatabase`:

- `customer_table`
- `admin_table`
- `brand_table`
- `product_table`
- `cart_table`
- `order_table`
- `contact_us_table`

`product_table.image` stores image binary (BLOB) and pages output it using base64 data URLs.

## 4) End-to-end user flows

## A. Customer account flow

1. **Sign up** via `htdocs/log-in/signup.php`.
   - Inserts into `customer_table`.
2. **Sign in** through `htdocs/log-in/customersignin.php` -> POST to `customer_sign_check.php`.
   - Sets `$_SESSION['log_in']` when credentials match.
3. **Logout** via `htdocs/log-in/log_out.php`.

## B. Browse and discover products

1. **Search page** (`htdocs/files/search.php`)
   - Lists brands as categories.
   - Accepts POST search term and queries `product_table` with `LIKE`.
2. **Category page** (`htdocs/files/Categories/categories.php`)
   - Filters products by `brand_id` from `?id=`.
3. **Product page** (`htdocs/files/Products/product.php`)
   - Shows product details/specs and an "Add To Cart" action.

## C. Cart and ordering

1. **Add to cart**
   - `product.php` links to `cart/cart_updating.php?id=<product_id>`.
   - Insert into `cart_table(customer_id, product_id)`.
2. **Cart view** (`cart/cart.php`)
   - Joins `cart_table` and `product_table` for current user.
   - Provides "Place Order" and "Remove" links.
3. **Place order** (`order/order.php?id=<product_id>`)
   - Inserts into `order_table(order_product, order_customer)`.
4. **Remove from cart** (`order/order_delete.php?id=<product_id>`)
   - Deletes row from `cart_table` for current user/product.

## D. Contact flow

- `htdocs/files/Contact.php` accepts message title/body and inserts to `contact_us_table` tied to logged-in customer.

## E. Admin flow

1. **Admin sign in** (`htdocs/log-in/adminsignin.php`) sets `$_SESSION['admin_in']`.
2. **Dashboard** (`htdocs/files/Admin/admin.php`) provides:
   - Brand list + add/update/delete forms.
   - Product create form.
   - Product list with edit/delete links.
   - Customer message list.
3. Action endpoints:
   - Brand: `add_brand.php`, `update.php`, `delete.php`
   - Product: `add_product.php`, `edit.php`, `update_product.php`, `delete_product.php`

---

## 5) Data model relationships

- `product_table.brand_id -> brand_table.brand_id`
- `cart_table.customer_id -> customer_table.customer_id`
- `cart_table.product_id -> product_table.product_id`
- `order_table.order_customer -> customer_table.customer_id`
- `order_table.order_product -> product_table.product_id`
- `contact_us_table.customer_id -> customer_table.customer_id`

The SQL dump defines indexes and foreign keys for several of these relations.

---

## 6) Notable implementation characteristics

- Inline SQL and mixed PHP/HTML in nearly every page.
- Minimal separation of concerns (no service/repository layers).
- Frequent reuse of header/footer markup by copy/paste rather than shared includes.
- Uses Windows-style relative paths (`..\..\`) throughout.
- Database connection is opened once by include and often `mysqli_close()` is called multiple times in the same request.
